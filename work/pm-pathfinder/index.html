A guided decision framework for product managers navigating ambiguity

PM Pathfinder
2025

The brief

Product managers spend a lot of time in grey areas. Should we build this feature? Is this bug a P0 or a P2? Do we do the technically right thing or the thing users are asking for?

PM Pathfinder is an interactive decision tool — you answer a series of questions about your situation and it surfaces the right framework, the right questions to ask, and the right stakeholders to loop in.

Not another "prioritisation matrix." It adapts to the type of decision you're actually making — scope, resourcing, technical debt, user research, and stakeholder alignment all have different paths.
React TypeScript Framer Motion Vercel Tool

How it works

01
Describe the type of decision you're facing — build vs buy, prioritisation, scope reduction, stakeholder conflict, etc.
02
Answer 3–5 targeted questions about your constraints: timeline, team size, user impact, reversibility.
03
Get a recommended framework with a worked example, a list of questions to bring to your next meeting, and links to deeper reading.

Technical approach

A decision tree engine at the core — each node is a question with weighted branches. Built in React with Framer Motion for transitions. The full decision tree lives in a JSON file — easy to extend without touching the UI code.

Zero backend. Everything runs client-side. Loads in under 600ms.
